**Files:**

- Reference: `Senate-PSI-Majority-Staff-Interim-Report-April-29-2026-FINAL.pdf`

- [ ] **Step 1: Read the PDF systematically and produce one citation record per footnote**

For each footnote in the report (numbered 1, 2, 3, ... through whatever the highest is), produce a record:

```json
{
  "id": "fn-N",                         // N matches the footnote number
  "bates_ids": ["PSI-HHS-..."],         // every Bates ID mentioned in the footnote text, in order
  "footnote_text": "...",               // the verbatim text of the footnote (cleaned of line breaks)
  "document_id": null,                  // always null in M1 (no documents posted yet)
  "external_url": null,                 // populate if the footnote contains an http(s) URL
  "emphasis_note": "(emphasis added)"   // only if the footnote contains this phrase; else omit
}
```

Rules:
- The footnote ID is `fn-N` where N is the literal footnote number from the PDF (NOT a sequential index).
- `bates_ids` lists every Bates ID found in the footnote text in source order. Patterns: `PSI-HHS-\d{9}(-\d+)?` and `PSICOVID_\d{8}(-\d+)?`.
- `footnote_text` is the verbatim text. Strip extra whitespace; preserve everything else.
- `document_id` stays null for ALL citations in M1 (we don't have documents yet — that's M2).
- `external_url` captures the FIRST http(s) URL in the footnote, if any. If a footnote has multiple URLs, list the most identifying one (usually the journal article or hsgac.senate.gov page; FOIA download links can be deferred).
- `emphasis_note`: only set if the footnote literally contains "(emphasis added)" or "(emphasis in original)".

The result is one large array in `src/content/citations.json`.

- [ ] **Step 1: Write a build-time helper that reads the glossary and emits the alias list**

`src/lib/glossary-aliases.ts`:

```typescript
import { readdirSync, readFileSync } from 'node:fs';
import { join } from 'node:path';
import { parse as parseYaml } from 'yaml';

interface AliasEntry {
  canonical: string;
  alias: string;
}

const GLOSSARY_DIR = './src/content/glossary';

export function loadGlossaryAliases(): AliasEntry[] {
  const entries: AliasEntry[] = [];
  const files = readdirSync(GLOSSARY_DIR).filter((f) => f.endsWith('.md'));
  for (const filename of files) {
    const path = join(GLOSSARY_DIR, filename);
    const content = readFileSync(path, 'utf-8');
    const fmMatch = content.match(/^---\n([\s\S]*?)\n---/);
    if (!fmMatch) continue;
    const fm = parseYaml(fmMatch[1]);
    const canonical = (fm.term as string).toLowerCase();
    entries.push({ canonical, alias: fm.term });
    if (Array.isArray(fm.aliases)) {
      for (const a of fm.aliases) entries.push({ canonical, alias: a });
    }
  }
  return entries;
}
```

- [ ] **Step 1: Generate a runtime lookup**

The simplest approach: emit a small JSON of `{term: short_def}` at build time, ship it with the site, and have a tiny script attach the popover content on page load.

`src/lib/glossary-shortdefs.ts`:

```typescript
import { readdirSync, readFileSync, writeFileSync, mkdirSync } from 'node:fs';
import { join } from 'node:path';
import { parse as parseYaml } from 'yaml';

const GLOSSARY_DIR = './src/content/glossary';
const OUT_PATH = './public/glossary-shortdefs.json';

export function emitShortdefs(): void {
  const map: Record<string, string> = {};
  const files = readdirSync(GLOSSARY_DIR).filter((f) => f.endsWith('.md'));
  for (const filename of files) {
    const content = readFileSync(join(GLOSSARY_DIR, filename), 'utf-8');
    const fmMatch = content.match(/^---\n([\s\S]*?)\n---/);
    if (!fmMatch) continue;
    const fm = parseYaml(fmMatch[1]);
    const canonical = (fm.term as string).toLowerCase();
    if (typeof fm.short_def === 'string') map[canonical] = fm.short_def;
  }
  mkdirSync('./public', { recursive: true });
  writeFileSync(OUT_PATH, JSON.stringify(map, null, 2));
}
```
